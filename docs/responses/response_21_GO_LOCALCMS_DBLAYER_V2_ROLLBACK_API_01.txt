CONTRAT API ROLLBACK RETENU
----------------------------

Méthode    : POST
URL        : /api/installer/rollback
Body       : {"module_id": "my_module"}

Réponse 200 :
  {
    "result":         "ok",
    "module_id":      "my_module",
    "backup_used":    "my_module_20260429T143000000000",
    "restored_files": ["my-module.js"]
  }

Erreurs :
  400 — module_id invalide (hors [a-z0-9_]+)
  403 — path violation (backup hors BACKUP_DIR)
  404 — aucun backup trouvé pour ce module_id
  500 — erreur de copie

Cas refusés :
  module_id "../etc"     → 400 (VALID_ID_RE bloque avant résolution)
  module_id "mod/evil"   → 400
  module_id ""           → 400
  module inexistant      → 404
  backup hors BACKUP_DIR → 403

GARANTIES DE SÉCURITÉ
----------------------

1. module_id → VALID_ID_RE = r"^[a-z0-9_]+$"
   Interdit .., /, \, espaces, tirets, majuscules
2. backup_src → backup_src.resolve().relative_to(BACKUP_DIR.resolve())
   Confinement dans install-backups/ vérifié
3. target_path → TARGET_PATHS["modules_dir"] uniquement
   Jamais fourni par l'utilisateur
4. Aucun chemin absolu accepté dans module_id
5. Aucun subprocess, aucun os.system
