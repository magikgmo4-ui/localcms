3. ÉLÉMENTS DE CONTEXTE — ÉTABLI
══════════════════════════════════════════════════════════════

[ÉTABLI] Le dépôt canonique versionne localcms-v5.html (front-end SPA).
[ÉTABLI] Le host FastAPI (main.py + run.*) est un serveur de développement local
         distinct du front-end : il expose des endpoints utilisés pendant le dev
         (proxy, webhook, fichiers statiques).
[ÉTABLI] Les fichiers host sont fonctionnels et utilisés en dev sandbox.
[ÉTABLI] .env.example ne contient pas de secrets ; il documente les variables
         d'environnement attendues.
[ÉTABLI] Aucun .gitignore canonique n'exclut explicitement ces fichiers
         (état observé au moment de l'audit).

══════════════════════════════════════════════════════════════
4. DÉCISION VALIDÉE
══════════════════════════════════════════════════════════════

Classe retenue : B — outillage durable séparé

Forme retenue :
  Versionner les fichiers host dans une branche dédiée distincte
  de la branche feature courante, afin de ne pas mélanger
  outillage de développement et évolutions du front-end SPA.

Détail :
  branche dédiée : tools/localcms-dev-host
  base : à confirmer avant GO_APPLY

Fichiers à inclure dans cette branche :
  - main.py
  - requirements.txt
  - run.bat
  - run.sh
  - .env.example

Fichiers à exclure (non concernés par cet arbitrage) :
  - localcms-v5.html et tout fichier front-end
  - tout fichier contenant des secrets réels

══════════════════════════════════════════════════════════════
5. POINTS À CONFIRMER AVANT GO_APPLY
══════════════════════════════════════════════════════════════

[À CONFIRMER] La base exacte de la branche tools/localcms-dev-host
              (main, develop, ou autre) doit être précisée par l'opérateur
              avant toute opération Git.

[À CONFIRMER] La réutilisation de ces fichiers host dans d'autres contextes
              (CI, staging, autre développeur) n'a pas été observée ;
              la pertinence du versionnement dépend de cet usage futur.

[HYPOTHÈSE]   Un README minimal dans la branche dédiée faciliterait
              la prise en main, mais n'est pas requis pour l'arbitrage.

══════════════════════════════════════════════════════════════

  est hors périmètre de cet arbitrage.
